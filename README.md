## Описание проекта
Данное приложение реализует управление пользователями и группами.  
Есть две основные страницы:
1. **Список пользователей** – таблица с возможностью поиска, сортировки и фильтрации по группам.
2. **Карточка пользователя** – просмотр и редактирование данных пользователя.  

Пользователи могут относиться к одной из нескольких групп, либо не иметь группы.  
Проект содержит серверную часть с REST API для CRUD операций и скрипт для бэкапа и восстановления данных.

## Стек технологий
- **Фронтенд**: React, Vite, TypeScript, MUI  
- **Бэкенд**: NestJS, TypeORM, SQLite 

- **Скрипт бэкапа**: Node.js 

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/JERR4/users-management-system.git
cd users-management-system
````

### 2. Установка зависимостей

**Фронтенд**:

```bash
cd frontend
npm install
```

**Бэкенд**:

```bash
cd backend
npm install
```

### 3. Настройка базы данных

- Используется SQLite, файл базы данных хранится в `backend/database.sqlite`.

Для заполнения базы данных начальными данными (400 пользователей и 5 групп) используется скрипт `seed.ts`:

```bash

cd backend
ts-node src/seed.ts
```

### 4. Запуск проекта

**Бэкенд**:

```bash
npm run start:dev
```

**Фронтенд**:

```bash
npm run dev
```

После запуска приложение доступно по адресу: `http://localhost:5173`

## Скрипт бэкапа и восстановления

Для резервного копирования и восстановления данных используется сервис `BackupService` и CLI-скрипт `backup.cli.ts`.

### Использование

Перейдите в директорию `backend` и выполните одну из команд:

#### Экспорт данных в файл
```bash
ts-node src/backup/backup.cli.ts export [путь_к_файлу]
````

По умолчанию данные сохраняются в `backup.json` в корне проекта.

#### Восстановление данных из файла

```bash
ts-node src/backup/backup.cli.ts restore [путь_к_файлу]
```

Если путь не указан, скрипт по умолчанию ищет `backup.json` в корне проекта.

### Формат данных

Файл резервной копии содержит JSON с двумя разделами:

```json
{
  "groups": [ /* список групп */ ],
  "users": [ /* список пользователей */ ]
}
```
